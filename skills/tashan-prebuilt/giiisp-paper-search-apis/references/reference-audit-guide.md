## 阶段一：内部一致性与交叉匹配

### 1. 正文引用与参考文献列表交叉匹配

**检查项**：
- [ ] 正文中每一个 `[作者, 年份]` 引用都在参考文献列表中有对应条目
- [ ] 正文中每一个 `[编号]` 引用（如 [1], [2-5]）都在列表中有对应条目
- [ ] 参考文献列表中未被正文引用的条目已标记为"多余"
- [ ] 正文引用与列表条目的作者拼写一致
- [ ] 正文引用与列表条目的年份一致

**常见错误**：
- 正文中引用 `(Smith, 2020)`，列表中为 `Smith, J. (2019)` — 年份不匹配
- 正文中引用 `[12]`，列表中仅 11 条 — 编号超出范围
- 列表中有条目但正文从未引用 — 多余条目

---

### 2. 引用格式一致性

**APA 7th 检查项**：
- [ ] 期刊文章：作者. (年份). 标题. *期刊名*, 卷(期), 页码. https://doi.org/xxxxx
- [ ] 书籍：作者. (年份). *书名*. 出版社.
- [ ] 会议论文：作者. (年份). 标题. In *会议名称* (页码). 出版社.
- [ ] DOI 前缀为 `https://doi.org/`
- [ ] 标题仅首单词首字母大写（冒号后首单词也大写）
- [ ] 期刊名斜体，卷号斜体，期号不斜体

**GB/T 7714-2015 检查项**：
- [ ] 期刊：[序号] 作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.
- [ ] 专著：[序号] 作者. 书名[M]. 出版地: 出版社, 出版年: 起止页码.
- [ ] 序号使用 `[1]` 格式，按引用顺序排列

**MLA 9th 检查项**：
- [ ] 作者. "标题." *期刊名*, vol. 卷, no. 期, 年份, pp. 页码.
- [ ] 标题使用引号，期刊名斜体

---

### 3. 作者姓名与缩写标准化

**检查项**：
- [ ] 全文中作者姓名格式一致（全名 / 姓+名首字母 / 仅姓）
- [ ] "et al." 使用符合格式要求（APA: >=3 作者；MLA: >=3 作者）
- [ ] 同一作者在不同条目中拼写一致
- [ ] 不猜测缺失的首字母；标记为"信息缺失"

**示例不一致**：
- `Zhang, Wei` vs `Zhang, W.` vs `W. Zhang` — 需统一
- `Li, Ming` vs `Lee, Ming` — 可能为拼写错误

## 阶段二：元数据与技术校验

### 4. DOI 与 URL 链接校验

**检查项**：
- [ ] DOI 格式为 `10.xxxx/xxxxx` 或 `https://doi.org/10.xxxx/xxxxx`
- [ ] DOI 中没有空格或非法字符
- [ ] URL 使用 `https://` 前缀（非 `http://`）
- [ ] URL 中没有追踪参数（如 `?utm_source=...`）
- [ ] arXiv URL 格式为 `https://arxiv.org/abs/xxxx.xxxxx`

**常见错误**：
- `doi: 10.1234/example` — 缺少 `https://doi.org/` 前缀
- `www.example.com` — 缺少协议前缀
- `https://arxiv.org/pdf/2301.00001.pdf` — 应使用 `/abs/` 而非 `/pdf/`

---

### 5. 期刊名称与缩写核查

**检查项**：
- [ ] 同一期刊在全文中使用统一名称
- [ ] 期刊缩写符合 ISO 4 或 NLM 标准（如指定）
- [ ] 期刊名大小写正确（如 *Journal of Machine Learning Research*）
- [ ] 不使用过期的期刊旧名（除非引用的是旧名发表的文章）

**常见不一致**：
- `Proceedings of the National Academy of Sciences` vs `PNAS` vs `Proc. Natl. Acad. Sci.`
- `Neural Information Processing Systems` vs `NeurIPS`

---

### 6. 出版年份与卷期逻辑核查

**检查项**：
- [ ] 年份在合理范围内（通常 1900-当前年）
- [ ] 同一期刊同一卷号不出现冲突年份
- [ ] 卷号 >= 期号（通常成立，非绝对）
- [ ] 页码为合理范围（起页 < 止页）
- [ ] 在线优先发表（Online First）的条目标注正确

**逻辑冲突示例**：
- `Nature, 2020, 580(7802)` 和 `Nature, 2019, 580(7803)` — 同一卷号跨年份需核实
- `pp. 150-120` — 起止页码颠倒

---

### 7. 参考文献完整性

**检查项**：
- [ ] 期刊文章：作者、标题、期刊名、年份、卷(期)、页码 — 缺一不可
- [ ] 书籍：作者/编者、书名、出版社、出版地、年份 — 缺一不可
- [ ] 会议论文：作者、标题、会议名、年份、页码 — 缺一不可
- [ ] 网页：作者（如有）、标题、网站名、URL、访问日期 — 缺一不可
- [ ] "In Press" / "Forthcoming" 条目已标注并注明来源

**缺失标记示例**：
- `[3] Smith, J. (2020). Title. Journal Name.` — 缺少卷期页码
- `[5] Author. (2021). Title.` — 缺少期刊名和页码

## 阶段三：伦理与程序审计

### 8. 冗余与重复参考文献检测

**检查项**：
- [ ] 同一篇论文不出现两次及以上
- [ ] 同一篇论文的不同版本（预印本 vs 正式发表）已区分或合并
- [ ] 作者、年份、标题高度相似（>80%）的条目已标记

**重复类型**：
- 完全重复：元数据完全一致
- 近似重复：标题/作者相同，但期刊/年份/页码有差异（可能是不同版本）
- 部分重复：同一作者同年多篇，标题相似（需核实是否为不同论文）

---

### 9. 自引与引用偏差评估

**统计项**：
- [ ] 主要作者自引数量 / 总引用数量 = 自引率
- [ ] 同一期刊引用数量 / 总引用数量 = 期刊集中度
- [ ] 同一研究团队引用数量 / 总引用数量 = 团队集中度
- [ ] 近 5 年文献占比（反映时效性）

**输出格式**：
```
自引率: 3/30 = 10%
期刊集中度: 5 篇来自同一期刊 (16.7%)
团队集中度: 4 篇来自同一研究团队 (13.3%)
近 5 年文献: 22/30 = 73.3%
```

**注意**：仅提供客观统计，不做道德评判。高自引率不一定有问题（如综述文章）。

---

### 10. 字母与年代顺序审计

**字母顺序检查（APA / MLA）**：
- [ ] 按第一作者姓氏字母顺序排列（A-Z）
- [ ] 同一作者多部作品按年份升序排列
- [ ] 同年同作者按标题字母顺序排列（a, b, c 标注）

**编号顺序检查（GB/T 7714）**：
- [ ] 按正文引用顺序编号 [1], [2], [3]...
- [ ] 同一位置引用多篇按编号升序排列 [2, 5, 8]

**常见错误**：
- `Smith (2020)` 在 `Adams (2019)` 之前 — 字母顺序错误
- `Smith (2020a)` 在 `Smith (2020b)` 之后 — 同年子排序错误

## Giiisp API 联合验证细则

### 验证触发条件

仅在以下情况触发 API 验证：
1. 审计过程中发现元数据可疑（标题/作者/年份不一致）
2. 参考文献条目包含 arXiv 编号或明确为 arXiv/OA 论文
3. 用户主动要求验证某条目的准确性

### 验证流程

```
审计发现异常条目
    |
    v
判断是否为 arXiv / OA 论文
    |
    +---> 是 arXiv
    |         v
    |     提取 arXiv 编号
    |         v
    |     调用 searchArxivByArxivNo1
    |         v
    |     比对 title / authors / year / abstract
    |         v
    |     生成比对报告
    |
    +---> 是 OA 磁盘阵列
    |         v
    |     提取论文标题
    |         v
    |     调用 searchArticlesByQuery1
    |         v
    |     比对标题和摘要匹配度
    |         v
    |     生成比对报告
    |
    +---> 不是 arXiv/OA
              v
          标记为"无法 API 验证，需人工核实"
```

### API 验证结果处理

- **一致**：标记为"API 验证通过"
- **不一致**：列出差异字段，标注"API 验证建议修正"
- **API 无结果**：标记为"API 未收录，请人工核实"
- **API 异常**：标记为"API 调用失败，请稍后重试或人工核实"

## 评分标准

### 整体评分

| 维度 | 权重 | 满分 | 评分标准 |
|------|------|------|----------|
| 交叉匹配一致性 | 20% | 20 | 无缺失/多余引用 |
| 格式合规性 | 15% | 15 | 格式错误 <= 5% |
| 姓名标准化 | 10% | 10 | 无姓名不一致 |
| DOI/URL 正确性 | 10% | 10 | 无格式错误 DOI/URL |
| 期刊名称一致性 | 10% | 10 | 无期刊名称冲突 |
| 年份卷期逻辑 | 10% | 10 | 无逻辑冲突 |
| 元数据完整性 | 10% | 10 | 无缺失关键字段 |
| 冗余检测 | 5% | 5 | 无重复条目 |
| 自引偏差 | 5% | 5 | 自引率 < 20% |
| 排序正确性 | 5% | 5 | 排序完全正确 |

### 等级划分

| 总分 | 等级 | 说明 |
|------|------|------|
| 95-100 | 优秀 | 可直接投稿 |
| 85-94 | 良好 |  minor 修正后投稿 |
| 70-84 | 合格 | 需修正明显错误 |
| < 70 | 不合格 | 需全面审查 |
