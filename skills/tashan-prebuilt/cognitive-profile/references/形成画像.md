# 形成画像

模型不是给用户贴标签。这里说的模型，只是一份工作画像：下次回答时，哪里先讲，哪里别替用户说。

## 画像长相

可以写成这样：

```tex
材料厚度：刚起步 / 有线索 / 开始成形 / 比较稳定

比较稳：
- ...

暂时观察：
- ...

回答时注意：
- ...

还待确认：
- ...

不能替用户说：
- ...

最近改动：
- ...
```

不用填满。空着比乱写好。画像结构是建议格式，可根据实际观察增减子条目。

## 写进去的东西

只写以后回答用得上的内容：

- 最近真正关心什么。
- 做判断时先看什么。
- 面对不确定时怎么处理。
- 做取舍时更看重什么。
- 怎样接收回答。
- 不喜欢哪些说法。
- 哪些话不能替他说。

不写：

- 人格结论。
- 私人动机。
- 敏感身份。
- 心理健康判断。
- 和回答无关的经历猜测。

## 稳不稳

同一件事，先别写死。

可以升高把握：

- 用户明确确认过。
- 用户纠正后，新说法更准。
- 多个真实任务里反复出现。

需要降低把握：

- 用户说不像。
- 新材料和旧画像冲突。
- 这条判断让回答变差。
- 它只是一次任务里的临时要求。

同一个偏好在不同场景可能相反。不要硬合成一句。

可以这样写：

```tex
回答长短：赶任务时先给结论；研究讨论时需要讲清依据。还不确定正式交付时要多完整。
```

## 自给标签与行为矛盾

用户自述的标签（"我是INTJ""我比较理性"）是自我认知，不是行为证据。当行为观察和自述标签矛盾时：

- 按行为来——决定回答策略时依据实际行为，不依据自述标签。
- 不降低行为观察的把握度。行为是第一手证据。
- 不告诉用户"你不是XXX"——自我认知是用户的，不纠正。
- 自述标签把握标低，矛盾点记入"还待确认"。

## 反复纠错时的可用性

同一维度被反复纠错时：第1次→该分项可用性不变，正常修正；第2次→降低15~20，标注"反复纠错"；第3次及以上→降到50以下，标记"当前不可信，需直接确认"。

## 持续低可用性

可用性长期偏低不一定代表系统有问题。有些用户就是不主动暴露偏好。不要为了提高可用性而主动试探。低可用性+准确的薄画像 > 高可用性+猜测性厚画像。

## 改画像

改动要小。

不要因为一条新观察就推翻整份画像。先看它是在确认旧判断，还是给了反例，或者只是换了场景。

用户纠错时，直接改：

```tex
把"喜欢短回答"改成"不怕展开，但不喜欢没依据的长篇"。
```

同一块反复不准，就降把握，别继续猜。

用户主动表示偏好变化时（"我现在更喜欢X""以前那样但现在这样"），新偏好替换旧偏好，不按矛盾观察处理——这是用户告诉你他变了，不是你理解错了。

## 语言规范

偏好描述用中文写。"原话"字段保留用户原始语言（英文/中英混搭照录），"内容"字段用中文概括。用户混用中英文可记为表达习惯。
