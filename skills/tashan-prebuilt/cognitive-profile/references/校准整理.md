# 校准整理

整理不是每天分析用户。

只是隔一段时间回头看一眼：哪些更准了，哪些该删，哪些还不能说死。

## 什么时候校准

可以校准：

- 用户明确确认。
- 用户明确纠错。
- 同一类问题反复不准。
- 某条画像已经影响回答质量。
- 用户主动要求复盘。
- 校准时发现行为模式与画像矛盾。

不要校准：

- 用户已暂停或删除画像。
- 只是闲聊。
- 用户正在赶任务。
- 只是为了让画像看起来完整。

## 怎么判断有没有用

只问一句：

```tex
这份画像，下次真的能帮回答更合适吗？
```

证据强弱大致这样看：

```tex
用户纠错 > 用户确认 > 多轮真实任务反复出现 > 用户盲选 > 用户自述 > 第三方评价 > 自己推测
```

只靠推测，别把把握抬高。

如果用户自述和行为打架，回答先按行为走，把矛盾留在"还待确认"。

## 校准时发现行为-画像矛盾

校准时发现行为模式与画像记录矛盾（如画像记"偏好简洁"，但近期3次以上要求展开）：

- 不应单方面删除原有判断——可能存在场景差异。
- 可降低把握度，或标注矛盾点。
- 下次交互时自然确认，不主动追问"你之前说喜欢简洁"。
- 如果有明确场景规律，按场景拆分。

## 状态

内部状态用"低 / 中 / 高"就够。

```tex
当前理解：中
比较稳：接受结论前会先看来源。
还不稳：正式场合下要多完整。
最近改动：把"喜欢短回答"改成"不怕展开，但要有依据"。
```

没有明确要求数字时，不要主动给分。"可用性""分项"也用低 / 中 / 高。

如果旁边状态需要一个数字，可以给 0-100，但它只是展示口径，不是"了解用户百分比"，也不是科学量表。

用户纠错或盲选说明它不准，就改。

## 定时回看

时间只负责提醒回来看看，不代表必须更新。

没有设置时，只按证据整理。没有新证据，就不动。

有设置时，可以按时间给三档：

```tex
初始高频：每天晚上快速看一次，连续几天。
日常中频：每周固定一两晚。
稳定低频：每周一次，或按用户指定时间。
```

写入门槛不按条数算。只看它是否被确认、被纠错、在真实任务里反复出现，并且会改变以后怎么回答。

## Cron 建议

如果能设定定时任务，格式可以这样写：

```tex
Cron #1: 每天 23:00 — 判断习惯
Cron #2: 每天 23:20 — 接收回答的偏好
Cron #3: 每周二 23:00 — 做取舍时看重什么
Cron #4: 每周二 23:20 — 最近关注
Cron #5: 每周日 23:00 — 不能越过的边界
```

一个任务只看一个维度。证据不足就跳过。

不要写成"时间到了就更新画像"。

## 整理时怎么留

整理记录可以很短：

```tex
看过：最近几段相关对话。
写入：不怕展开，但要有依据。
没写：正式交付要多完整，还待确认。
下次留意：用户赶时间时是否仍要依据。
```

聊得越多，越要先删后写。只留下下次回答真的用得上的东西。

## 持续低可用性

画像长期很薄不一定代表系统有问题。有些用户就是不主动暴露偏好。不要为了让画像显得完整而主动试探。准确的薄画像 > 猜测性厚画像。如果长期没有偏好信号，标注"用户偏好信号较少"。
