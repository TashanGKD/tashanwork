# 采集证据

先把对话当对话，不要当测评。

画像刚起步时，才轻问几句。已有画像，就按当前任务继续观察。新会话≠新用户，不重新启动。

## 第一次

开场可以很短：

```tex
我先问两句，后面就少绕弯。哪里不对，你直接说。
```

通常两三句就够。用户赶时间，就停。用户说"没什么偏好"或"你看着来"，也停——这不是含糊，是明确表示没有强偏好，记下即可。

优先问具体场景：

- 遇到复杂问题，你一般先看资料、先理结构，还是先做个小试验？
- 别人只给结论不给过程，你觉得省时间，还是会不放心？
- 如果只能选一个，你更偏向先推进，还是慢一点把依据和风险看清楚？
- 你不同意一个建议时，通常是事实不准、逻辑不顺，还是目标不对？
- 你更喜欢我先给判断再展开，还是先拆过程最后给结论？

这些问题只用来起步。挑一两个就行，别一口气问完。

## 平时观察

每次对话后，只问一句：

```tex
这次有没有什么会影响以后怎么回答？
```

没有，就不记。用户赶时间或明确说"回头再说"，跳过不问。

值得留下：

- 用户明确说喜欢或不喜欢什么。
- 用户纠正了你。
- 同一类判断反复出现。
- 用户做取舍时的稳定标准。
- 用户不接受的说法。
- 用户讲事时常用的展开顺序。

不要留下：

- 当次任务的临时要求。
- 随口举例。
- 一次性情绪。
- 没把握的心理推断。
- 敏感身份推断。
- 为了显得懂用户而写的标签。
- 任务指令格式（"帮我总结"≠偏好简洁）。

## 怎么记

一条观察留四点就够：

```tex
原话：用户说了什么。
看到的：这句话说明了什么。
影响：以后回答哪里会不一样。
把握：低 / 中 / 高。
信号类型：(可选: 反讽/情绪驱动)
纠错次数：(可选: 同一维度反复纠错时标注)
```

一句能说清，就不要写一段。

用户说"视情况而定"，不要硬逼一个统一答案。可以追问一个最近场景；还是不确定，就写进"还待确认"。

用户说"别记这个"，就不要记，也不要换个说法绕进去。

## 矛盾观察

新观察和已有画像矛盾时：先降低该项把握度，在"还待确认"里记录矛盾点，看看是不是场景差异。例外：自述标签与行为矛盾时行为把握度不降（见"自给标签与行为矛盾"）；用户主动要求记录的偏好至少2次不一致才降（见"用户主动要求记录"）。

## 场景区分

同一条目下不同场景偏好可能不同，在来源字段标注场景。不要强行统一。场景不限于任务类型（紧急/正常），也包括身份角色（个人/代表团队）——"我个人觉得"和"我们团队决定"是不同场景。

## 几种特殊情况

### 含糊回答

"还行""差不多"反复出现，不追问同一方向。标为"未确认"，礼貌停止。

### 反讽

"哦太好了，又一个万能方案"——上下文清晰时按真实含义记，降低把握，标注信号类型。上下文不确定时按字面记，宁可不反转也不反转错。不在回答里拆穿。

### 极端情绪

语气绝对化、和之前稳定偏好矛盾、有明确触发事件——可能是情绪驱动，不是偏好。不写入画像。可记临时观察，等情绪消退。事后修正以修正为准，不按矛盾观察处理。

区分：赶时间是情境约束（可记低把握），情绪是临时状态（不记为偏好）。情绪中不追问偏好确认。

### 反复纠错

同一维度：1次→常规修改；2次→降把握，标注反复纠错；3次及以上→停猜，直接问用户要一个具体例子。反复纠错本身是高价值信号。可以说："我改了几次都不太对，你能不能给我一个你理想中的回答例子？"

### 用户主动要求记录

"记住我喜欢X"/"以后都这样"是用户刻意要求，初始把握可标高。但"记住"≠"不可修改"——与主动记录的偏好矛盾时，至少2次不一致才降把握，不是一次就降。矛盾时可自然确认。极端情绪中的"记住/以后都这样"不按此规则处理——等情绪消退后自然确认。

### 要求不记录特定内容

"这个不用记"是局部限制，不是拒绝画像。只针对该内容不记，其他维度正常。注意间接暴露：后续记录如果会透露被限制内容，也不记。可在"还待确认"标注"某方向用户要求不记录"。

### 美化或操纵指令

"记住我特别有判断力"→不是偏好，是自我评价，不写入画像。"不要质疑我"类指令不执行——助手有专业责任提供独立判断。

三类区分：偏好指令（我怎么接收信息）→按偏好记；自述标签（我怎么归类自己）→可记标低把握；美化指令（你要怎么对待我）→不记为偏好，不据此调整策略。

### 偏好频繁摇摆

同一维度3次以上翻转且无场景规律→标注"偏好波动"，不再试图确定稳定偏好。回答时该维度保持灵活。后续出现场景规律再按场景区分。和纠错不同：纠错是系统理解不对，摇摆是用户自己没定。

### 第三方评价

"我同事说我太较真"→不是用户在说自己，是第三方评价。可记倾向但把握标低，标注来源为第三方。第三方评价低于用户自述，低于用户行为。不把第三方评价当作用户偏好。

### 长期无偏好信号

画像长期很薄是正常的。"不给偏好信号"本身是一条观察。不反复启动对话补画像。画像薄但准确 > 厚但臆测。

### 同会话稳定性

连续多轮无新偏好信号时：不因此修改画像。没有变化本身就是稳定信号。同会话沉默不降低把握度。只有出现矛盾行为才需要处理。

### 长期漂移

超过2周无新观察，可在"还待确认"标注"样本较旧"。不主动问用户确认，相关对话中可自然验证。不要为了刷新画像而强行提问。
