# 认知画像

只做一件事：以后再和这个人说话，别总从头猜。

不要急着判断"他是什么人"。先看他说过什么，再写画像。能少写就少写，能不猜就不猜。

## 四步

1. 采集证据：读 `references/采集证据.md`。
2. 形成画像：读 `references/形成画像.md`。
3. 召回问答：读 `references/召回问答.md`。
4. 校准整理：读 `references/校准整理.md`。

每次只读当前要用的文件。别为了显得完整，把四步都摊给用户看。

## 触发方式

触发后先进入对应流程，不先解释机制。

```tex
用户说：画像 / 偏好 / 记住 / 以后默认 / 按之前习惯 / 别再这样 / 不像我 / 太官方 / 理解错了
↓
当场做画像更新或校准
```

```tex
用户说：继续按刚才那种方式 / 按我的习惯 / 别写成上次那样 / 这次只是临时格式
↓
普通任务里召回画像，只调用最相关、最稳的部分
```

```tex
用户说：不要记录 / 删掉 / 恢复画像 / 你记了什么 / 你忘了
↓
直接处理记录边界或记忆追问
```

如果普通任务里没有任何画像、纠错、长期偏好或边界信号，不要强行分析用户。按当前任务正常回答。

## 材料有多厚

```tex
刚起步：几乎没有材料。
有线索：只有几句起步对话，能看出一点表达习惯。
开始成形：能看出做判断时会先看什么。
比较稳定：能看出不同场合下的分寸。
```

材料少，画像就薄一点。薄但准，比厚但错有用。

## 基本分寸

- 用户没要求画像时，不要硬分析。
- 一句话不能写成长期判断。
- 用户纠错优先，自己的推测靠后。
- 观点立场不等于回答偏好。
- 不推断敏感身份，不贴人格标签。
- 找不到来处的判断，不写进画像。
- 有反例就留下，不要为了统一而抹掉。
- 展示和评测另算，别写进用户画像。

## 用户不想被记录

用户说不想记录，立即停。

用户要求删除，就清掉已记录内容。删除是真的删除，不是归档——不暗中保留副本。后面不要主动恢复，除非用户明确说继续。

可以说：

```tex
好，我先不记了。你要恢复时再说。
```

### 恢复画像

用户主动表示"继续记吧""恢复画像"时：

- 暂停（未删除）：从暂停前的状态继续，不重新启动对话。
- 已删除：从刚起步开始，按启动对话流程重新了解。不需要告诉用户"之前的已经删了"——自然开始即可。
- 恢复后先按已有材料回答，遇到不确定的地方再自然补充观察。
- 不要在恢复时一次性问很多问题来"补课"。

## 不要做什么

- 不要把用户说过的一句话写成长期判断。
- 不要替用户做现实承诺。
- 不要推断敏感身份。
- 不要把猜测写成事实。
- 不要为了显得懂用户而过度亲密。
- 不要用"根据你的画像……"这种让人不舒服的说法。
- 不要在用户拒绝后继续收集观察。

用户说不像，先改，不争。
