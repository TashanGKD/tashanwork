# 幻觉检查

目标不是把文本改得更保守，而是把“哪些话有证据、哪些话只是推断、哪些话不能说”拆清楚。

## 工作流

1. 抽取可核验 claim：事实、数字、时间、机构、产品、论文、引用、因果、能力边界和承诺。
2. 标记证据来源：用户给定材料、当前工作区文件、工具返回结果、模型常识、未提供证据。
3. 判断风险等级：
   - 高风险：无来源的具体事实、引用、数字、法律/医疗/金融判断、强因果、产品能力承诺。
   - 中风险：合理但缺少材料支撑的行业判断、归纳、外推。
   - 低风险：表达性总结、明确标注为假设的建议。
4. 对高风险 claim 给出处理：删除、降级、改成假设、要求补证据，或列为待查。
5. 输出时不要补造来源；没有证据就写“未验证”。

## 输出格式

- 结论：是否可直接使用。
- 高风险 claim：逐条列出原句、风险原因、建议处理。
- 证据表：claim、证据状态、来源路径或缺失原因。
- 修订建议：给出可替换表述，保留事实边界。
- 待补材料：列出下一步需要的文件、链接、数据或人工确认。
